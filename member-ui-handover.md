# 部材UI 専用引き継ぎ資料

> **この資料の目的**  
> モバイル側が部材一覧UIを「単なる部材名リスト」として推論実装しないための専用仕様書です。  
> Web版では部材が「継手の種別・ピン取り」という分類軸でグループ化されており、このグループ構造がユーザーの閲覧体験の核心です。

---

## 1. 部材UIの目的

部材一覧は「どの部位（部材）が、どの接合設計（継手）を使い、どの階層に存在するか」を確認するための画面です。

具体的には次の問いに答えます：

- **「大梁GB1はどんな継手を使っているか？」** → 継手名と、そのボルトサイズ・本数
- **「この部材は何階で使われているか？」** → targetLevels による階層表示
- **「仮ボルトは何を使うか？」** → 仮ボルトサイズ・本数（継手から計算）

### 継手一覧との違い

| | 継手一覧 | 部材一覧 |
|---|---|---|
| 主役 | 接合設計（ボルト本数の根拠） | 実際の部位（発注時の呼び名） |
| 行の単位 | 継手1件 | 部材1件（または countAsMember な継手） |
| 追加情報 | なし | 「使用継手名」「対象階層」 |
| 編集後の影響 | 参照する全部材に波及 | その部材だけ |

部材は継手を「参照」しているため、継手が変われば部材のボルト本数も変わります。

### モバイルでの位置づけ

部材一覧は「注文明細の内訳確認」から逆引きするための補助画面です。「M22×60が96本？ → どの部材から来ているか？ → 大梁GB1が8箇所 × 12本」という確認導線を支えます。

---

## 2. 部材一覧と継手一覧の関係

Web版の部材一覧は、継手一覧と**同じグループ分類軸・同じグループ順序**を使っています。

継手一覧のセクションが「大梁 / 大梁(ピン取り) / 小梁 / ...」という順番なら、  
部材一覧のセクションも**同じ順番**で表示されます。  
この一致は偶然ではなく、コード上で同じ `type × isPinJoint` の組み合わせをハードコードされた配列で定義しているためです。

## 3. Web版における部材の並び順とグループ分け

### グループ（セクション）の一覧と固定順序

部材一覧は以下の順でセクションが並びます。データが存在するセクションだけが表示されます（空のセクションは非表示）。

| # | セクション見出し | 対象 type | isPinJoint | 色テーマ |
|---|---|---|---|---|
| 1 | 部材 - 大梁 | `girder` | false | 青 |
| 2 | 部材 - 大梁 (ピン取り) | `girder` | true | シアン |
| 3 | 部材 - 小梁 | `beam` | false | 緑 |
| 4 | 部材 - 小梁 (ピン取り) | `beam` | true | ティール |
| 5 | 部材 - 本柱 | `column` | false | 赤 |
| 6 | 部材 - 間柱 | `stud` | false | インディゴ |
| 7 | 部材 - 間柱 (ピン取り) | `stud` | true | 紫 |
| 8 | 部材 - 胴縁 | `wall_girt` | false | グレー |
| 9 | 部材 - 母屋 | `roof_purlin` | false | オレンジ |
| 10 | 部材 - その他 | `other` | false | アンバー |
| 11 | 部材 - その他 (ピン取り) | `other` | true | アンバー |

このリストは**ハードコードされた固定順序**です。ユーザーが変更することはできません。

### 継手一覧との順序比較

継手一覧と部材一覧のセクション順序は**ほぼ同じ**ですが、1点異なります。

| 順序 | 継手一覧 | 部材一覧 |
|---|---|---|
| 1〜4 | 大梁・小梁（同順） | 大梁・小梁（同順） |
| 5 | **間柱** | **本柱** |
| 6 | **間柱 (ピン取り)** | **間柱** |
| 7 | **本柱** | **間柱 (ピン取り)** |
| 8〜11 | 胴縁・母屋・その他（同順） | 胴縁・母屋・その他（同順） |

`column`（本柱）と `stud`（間柱）の順序が入れ替わっています。  
モバイルで実装する際は、**部材一覧側の順序**（本柱が間柱より前）を正として使ってください。

### セクション内の並び順（デフォルト）

ユーザーが列ヘッダーでソートしていない状態では、各セクション内の部材は以下の順で並びます：

1. **使用継手名の五十音順**（`joint.name` で localeCompare 'ja'）
2. 同じ継手を使う部材の中では、**部材名の五十音順**（`member.name` で localeCompare 'ja'）

このデフォルト順は「同じ継手を使う部材がまとまって見える」効果があります。例えば「GB1継手を使う部材（GB1-A, GB1-B, GB1-C）」が連続して並びます。

## 4. ピン取りグループの扱い

### 「ピン取り」とは

ピン接合（`isPinJoint: true`）の継手を使う部材です。梁端部の形状が特殊で、フランジのボルトを使わずウェブだけで接合します。

### 判定ロジック

部材がどのグループに属するかは、参照している**継手**の `isPinJoint` フラグで決まります。

```
部材 → 継手を参照（jointId） → 継手の isPinJoint を見る
  isPinJoint: false → 通常グループ（例: 大梁）
  isPinJoint: true  → ピン取りグループ（例: 大梁 (ピン取り)）
```

部材データ自体にはピン取りフラグはありません。参照先の継手フラグで決まります。

### 見出し表記

| 継手種別 | isPinJoint: false | isPinJoint: true |
|---|---|---|
| girder | 部材 - 大梁 | 部材 - 大梁 (ピン取り) |
| beam | 部材 - 小梁 | 部材 - 小梁 (ピン取り) |
| stud | 部材 - 間柱 | 部材 - 間柱 (ピン取り) |
| other | 部材 - その他 | 部材 - その他 (ピン取り) |
| column | 部材 - 本柱 | （ピン取りは存在しない） |
| wall_girt | 部材 - 胴縁 | （ピン取りは存在しない） |
| roof_purlin | 部材 - 母屋 | （ピン取りは存在しない） |

### ピン取りセクションでのテーブル列の違い

ピン取りグループのテーブル列はフランジを省略します。

**通常グループ（大梁・小梁）の列**:
```
部材名 / 使用継手 / フランジ(サイズ×本数) / ウェブ(サイズ×本数) / 仮ボルト(フランジ) / 仮ボルト(ウェブ)
```

**ピン取りグループ（大梁(ピン取り)・小梁(ピン取り)）の列**:
```
部材名 / 使用継手 / ウェブ(サイズ×本数) / 仮ボルト
```

フランジを使わないためフランジ列がありません。

## 5. 部材一覧で表示すべき項目

### 部材1行あたりの表示内容

| 項目 | 説明 |
|---|---|
| 部材名 | `member.name` |
| 対象階層バッジ | targetLevels の内容に応じて表示（後述） |
| 使用継手名 | `joint.name`（参照継手） |
| 継手カラーバッジ | `joint.color`（丸い色ドット） |
| ボルト情報 | 継手の種別・ピン取りに応じてフランジ/ウェブを表示 |
| 仮ボルト情報 | 壁面・母屋・柱以外は表示（計算値） |

### 継手種別ごとのボルト情報列

| 継手種別 | ピン取り | ボルト列 | 仮ボルト列 |
|---|---|---|---|
| girder / beam / other / stud | false | フランジ列 + ウェブ列 | 仮ボルト(フランジ) + 仮ボルト(ウェブ) |
| girder / beam / other / stud | true | ウェブ列のみ | 仮ボルト(ウェブ) のみ |
| column | false | エレクション列のみ | なし |
| wall_girt / roof_purlin | false | ボルトサイズ列のみ | なし |

### `countAsMember` な継手の扱い

継手に `countAsMember: true` が設定されている場合、その継手は**部材一覧にも登場**します。

- `member.isMember: false` として扱われる
- 部材名・継手名は同じ（継手名がそのまま部材名になる）
- 編集ボタンは「継手編集」モーダルへ遷移（部材編集モーダルではない）
- 対象階層バッジ: 常に「全」（緑バッジ）
- キューブアイコンがついて「部材として集計される継手」であることを示す

## 6. targetLevels の表示ルール

### バッジの出し分け

部材行の部材名の右隣に階層バッジが表示されます。

| 条件 | 表示 | バッジ色 |
|---|---|---|
| `targetLevels` が空配列（`[]`）または未定義 | `全` | グレー |
| `targetLevels.length <= 3` | 階層ラベルを短縮して並べる（例: `2,3,R`） | 青 |
| `targetLevels.length > 3` | `{N}フロア` | 青 |
| `countAsMember` の継手（isMember: false） | `全` | 緑 |

### 短縮表示のルール

`targetLevels` に階層IDが2〜3件の場合、各IDを `getProjectLevels()` で変換してラベルを取得し、数字部分だけを抽出して `,` で連結します。

- simpleモード: `"2"` → `"2F"` → `"2"` として表示
- advancedモード: `"1F"` → そのままラベルとして表示

### フィルタータブ

部材一覧画面の上部に「全て / 2階 / 3階 / R階 / ...」のタブが表示されます（`state.activeMemberLevel`）。

フィルターの適用ルール:
- 「全て」選択: 全部材を表示
- 特定階層選択:
  - `countAsMember` な継手（isMember: false）→ 常に表示
  - `targetLevels.length === 0` → 全階対象なので常に表示
  - `targetLevels.includes(selectedLevel)` → 表示

## 7. モバイル向け再設計時の必須要件

以下を満たさない実装は「Web版の部材一覧とは異なる別物」になります。

### A. グループ分けは固定順序のセクションで行う

継手の種別（type）× ピン取り（isPinJoint）の組み合わせで、11種類のグループを固定順で表示します。ユーザーが「大梁の部材を一覧したい」「ピン取りだけ確認したい」ときに即座に見つけられるのはこのグループ構造があるからです。

**禁止**: 全部材をフラットなリストにして、部材名の五十音順でソートするだけの実装。

### B. グループの順序はハードコードで固定する

```
大梁 → 大梁(ピン取り) → 小梁 → 小梁(ピン取り) →
本柱 → 間柱 → 間柱(ピン取り) → 胴縁 → 母屋 →
その他 → その他(ピン取り)
```

この順序はユーザーが変更することも、Firestoreから動的に決まることもありません。

### C. グループの判定は参照継手の type と isPinJoint で行う

部材自身にグループ分類のフィールドはありません。必ず参照先の継手（`member.jointId` → `joint.type` + `joint.isPinJoint`）で分類してください。

### D. データが存在するグループだけ表示する

「大梁の部材が1件もない工事」では大梁セクションを表示しません。存在するグループだけを上から順に表示します。

### E. セクション内のデフォルト順は「継手名 → 部材名」の五十音順

同じ継手を使う部材がまとまって見える並び方です。単純な部材名の五十音順とは異なります。

### F. 対象階層フィルターを実装する

全て / 特定階層 のタブ切替を実装し、targetLevels によるフィルタリングを行います。

## 8. 部材詳細と継手詳細の導線

### 画面遷移の構造

```
部材一覧
  ├── 部材タップ → 部材詳細
  │     └── 使用継手タップ → 継手詳細
  └── 階層タブ → フィルター変更（同一画面）
```

### 部材詳細で見せるべき情報

| 項目 | 内容 |
|---|---|
| 部材名 | `member.name` |
| 使用継手名 | `joint.name`（タップで継手詳細へ） |
| 対象階層 | targetLevels の表示 |
| ボルト設計 | 継手のフランジ/ウェブのサイズ・本数 |
| 仮ボルト情報 | 計算値（継手の種別・ピン取り依存） |

### 継手詳細への遷移

部材詳細画面から「使用継手名」をタップすると継手詳細へ遷移します。  
**重要**: `countAsMember: true` の継手は部材一覧でも登場しますが、タップしたときは「部材詳細」ではなく「継手詳細」に直接遷移してください（部材固有の情報がないため）。

## 9. 実装者向け注意事項

### 注意 1: 部材一覧のグループ化は継手側のデータで決まる

部材データ（`member`オブジェクト）には `type` フィールドがありません。グループ分けは必ず `member.jointId` で継手を引き、`joint.type` と `joint.isPinJoint` を参照してください。

```
部材 → jointId → joints配列から検索 → joint.type + joint.isPinJoint でグループ決定
```

### 注意 2: isPinJoint は部材のフラグではなく継手のフラグ

「ピン取り部材」という概念はありません。「ピン取り継手を参照している部材」です。同じ継手を参照する部材は全員同じピン取りグループに入ります。

### 注意 3: column（本柱）にはピン取りグループが存在しない

`column` 型の継手に `isPinJoint: true` を設定することはできない設計です（フォームUI側で制御しています）。「本柱 (ピン取り)」というグループは存在しません。

### 注意 4: 部材一覧と継手一覧のセクション順序はほぼ同じだが「本柱」と「間柱」の順が逆

継手一覧: `... beam_pin → stud → stud_pin → column → ...`  
部材一覧: `... beam_pin → column → stud → stud_pin → ...`

モバイルで部材一覧と継手一覧を別画面で実装する場合、それぞれに適切な順序を使ってください。

### 注意 5: `countAsMember` な継手が部材一覧に混在する

プロジェクトによっては、継手に `countAsMember: true` を設定することがあります。この場合、その継手は部材一覧にも表示されます（isMember: false として）。一覧の見た目は通常の部材と同じですが、タップ時の遷移先が「継手詳細」になります。

### 注意 6: フィルタータブは部材の targetLevels によって効き方が変わる

```
targetLevels: [] (空配列) → 全階層対象 → どのタブでも表示
targetLevels: ["2", "3"]   → 2階・3階のみ → "2階"タブや"3階"タブでは表示、"R階"タブでは非表示
countAsMember な継手      → 全階層扱い → どのタブでも表示
```

`null` や `undefined` は `[]` と同じ「全階層」として扱います。

### 注意 7: ソート状態は部材一覧側では「継手名優先」がデフォルト

Web版のデフォルト並び順は「継手名の五十音順、次に部材名の五十音順」です。単純な部材名の五十音順ではありません。「同じ継手を使う部材がまとまって見える」ことが業務上重要です。
